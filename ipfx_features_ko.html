IPFX 추출 특성 설명

multiPFX 파이프라인에서 추출되는 47개 후보 특성 — QC 필터 통과 후 종별 35–42개가 차원 축소에 사용됨

QC 필터 — 차원 축소 전처리 단계

47개 후보 특성을 PCA / t-SNE / UMAP에 바로 넣지 않고, 두 단계의 결측치 기반 필터를 먼저 적용합니다. 이는 Allen Institute 참조 노트북(synphys_culture_classifier.ipynb, Cell 18)의 전처리 방식을 그대로 따른 것입니다.

Pass 1 — 행(cell) 필터
특성의 50% 이상이 있는 세포만 유지
전체 후보 특성 중 절반 이상이 NaN인 세포는 제거합니다. 예를 들어 chirp 스윕 실패 + AP 파형 추출 실패가 겹친 세포가 여기서 걸러집니다.
Pass 2 — 열(feature) 필터
80% 이상의 세포에서 추출된 특성만 유지
살아남은 세포들 중 20% 초과가 NaN인 특성은 해당 종의 분석에서 제외됩니다. 종마다 데이터 커버리지가 달라 사용 특성 수가 달라지는 이유입니다.
# Pass 1 — 세포 필터 (행)
cell_mask = df[features].notna().mean(axis=1) > 0.50

# Pass 2 — 특성 필터 (열)
feat_mask = df[features].notna().mean(axis=0) > 0.80

두 필터 이후에도 남아 있는 소수의 NaN은 KNNImputer(k=5)로 보간합니다. 이후 RobustScaler로 정규화한 뒤 PCA / t-SNE / UMAP을 실행합니다. 즉, 임베딩 단계에는 NaN이 전혀 들어가지 않습니다.
